path('study/') только GET
	Выводится список из 10 случайных проверенных Вопросов. User в режиме "обучение" должен на них ответить.
	
path('check/') GET - Вопрос, POST - Проверка
	Выводится 1 случайный непроверенный Вопрос для проверки.
	User проверяет его и высталяет flag - "true" либо "false". POST создает "Проверку".
	Логика Проверок:
	Если 3 "-"  то Вопрос "Отвергнут"(RJ), Если "+" на 3 больше чем "-"  то "Принят"(AC).
	т.е. (AC) может быть только при счете 5-2, 4-1, 3-0, тем самым кол-во "Проверок" не больше 7.
	
path('my/') - GET(list) POST, <pk>/ - GET(retrieve) PUT PATCH DELETE
	Работа с "Вопросом". Просмотр списка непроверенных (PR) и отвергнутых (RJ) "Вопросов". Создание "Вопроса".
	Удаление "Вопроса", только (PR) и (RJ).
	Изменение "Вопроса", только (RJ). При этом старые "Ответы" удаляются и создается новый массив "Ответов".
	Все "Проверки", по этому "Вопросу" тоже удаляются, процедура проверки начинается заново, "Вопрос" становится (PR).
